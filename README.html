🧠 01 — Philosophy & Workflow

The AI-First Brain Map

Everything in this template revolves around one idea: give the AI enough context and it will write great code on the first try.

🧠
Context is King
The AI works better with complete, organized context. CLAUDE.md + PRD + feature plans = better code on attempt #1.
📄
Document First
A good plan prevents bad implementations. Never ask the AI to code without a plan in .agents/plans/.
✅
Validate Every Step
Every task in a plan should have a validation command. Run it before moving to the next task.
🔁
Incremental Delivery
Break features into small, testable phases. Each phase delivers something a user can see and touch.

The Full Dev Cycle

Follow this cycle for every project and every feature. The loop repeats until the app is done.

1
Scaffold the AppPick your framework and set up the project: /init-project
2
Write the PRDDescribe your idea → AI generates .claude/PRD.md: /create-prd
3
Generate Project RulesAI analyzes codebase → writes CLAUDE.md: /create-rules then review & edit
4
Prime the AgentLoad full context at start of every session: /prime
↓   Feature Development Loop ↓
5
Plan the FeatureDescribe what you want → AI creates .agents/plans/feature.md: /plan-feature [description]
6
Review & Approve the PlanRead the plan. Check files to create, tasks, validation commands. Don't skip this.

ExecuteAI implements step by step: /execute .agents/plans/feature.md

9
CommitClean structured git commit: /commit
🛠️ 04 — Get Started

Starting a New Project

Follow these steps exactly when cloning this template for a new project.

Step 1 — Copy the Template

🪟 Windows (PowerShell)
git clone <template-url> my-new-app
cd my-new-app
Remove-Item -Recurse -Force .git
git init
🐧 Linux / macOS
git clone <template-url> my-new-app
cd my-new-app
rm -rf .git
git init

Step 2 — Scaffold Your Framework

# Next.js 15 (recommended for full-stack)
npx create-next-app@latest ./ --typescript --tailwind --app --src-dir --no-turbopack

# Vite + React (frontend only)
npx create-vite@latest ./ --template react-ts

# Python FastAPI (backend)
uv init && uv add fastapi uvicorn

Steps 3 – 4: PRD & Rules

/create-prd    # describe your idea → generates .claude/PRD.md
/create-rules  # analyze codebase → generates CLAUDE.md (then edit it!)

Feature Loop

/prime                                    # run at start of EVERY session
/plan-feature add user authentication     # → .agents/plans/add-user-auth.md
# ... review the plan ...
/execute .agents/plans/add-user-auth.md  # implements step by step
# ... validate manually ...
/commit                                   # clean git commit

🌐 06 — Cross-Platform Rules

Windows & Linux Compatibility

Follow these rules and the template will work without changes on both platforms.

✅
Delete lockfile on OS switch
Always delete package-lock.json before running npm install after switching OS. The lockfile encodes OS-specific binary paths.
❌
Never hardcode platform packages
Never add @rollup/rollup-linux-x64-gnu or similar to package.json. npm resolves the right native binary automatically.
🔡
Use forward slashes in paths
Forward slashes (/) work on both platforms in most tools. Avoid absolute paths like C:\Users\... in docs or plans.
📋
Provide both shell variants
When writing shell commands in plans or docs, always include both bash (Linux) and PowerShell (Windows) versions.

Shell Command Examples

🐧 Linux / macOS
cp .env.example .env.local
rm -rf .git
tree -L 3 -I node_modules
🪟 Windows (PowerShell)
copy .env.example .env.local
Remove-Item -Recurse -Force .git
Get-ChildItem -Recurse -Depth 3

📝 08 — Quick Start Checklist

New Project Checklist

Work through this list every time you start a new project from this template.

1
Copy template — remove old .git, run git init
2
Scaffold framework — run /init-project to pick and scaffold your stack
3
Describe your app — tell the AI what you want to build, run /create-prd
4
Generate rules — run /create-rules, then review and edit CLAUDE.md
5
First commit — git add . && git commit -m "chore: init project from template"
6
Prime the agent — run /prime at the start of every work session from now on
7
Plan first feature — /plan-feature [describe what you want]
8
Review the plan — open .agents/plans/feature.md and read it carefully
9
Execute — /execute .agents/plans/feature.md
10
Validate & commit — run tests + lint, check manually, then /commit
♻️ Repeat steps 6–10 for every new feature until the app is done
